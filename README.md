# Code Battle — 1v1 Real-Time Coding Arena

A competitive coding platform where players battle head-to-head in real-time, solving AI-generated programming problems to win trophies. Optional crypto betting via Ethereum smart contracts on Sepolia testnet.

## Overview

Code Battle is a full-stack real-time competitive coding platform. Two players are matched based on trophy count, given the same AI-generated coding problem, and race to solve it first. The winner earns trophies and can optionally wager ETH via a Solidity smart contract on Ethereum Sepolia testnet. Problems are dynamically generated by an AI agent (Lyzr) based on each player's skill level and chosen programming language. Code is executed and judged by Judge0.

## Features

### Authentication
- Email + password signup with OTP email verification
- JWT stored in httpOnly cookie (7-day expiry)
- Forgot password / reset password via OTP
- Protected routes — unauthenticated users redirected to login

### Dashboard
- Welcome screen with username and current arena
- Live trophy count and arena progress bar
- Win/loss/winrate stats
- Total battles fought
- Achievement badges gallery (earned automatically)
- Recent activity summary
- "Enter Battle" button linking to the game page

### Matchmaking
- Trophy-based matchmaking (±200 trophy range)
- Language selection per player (JS, Python, Java, C++)
- Optional trophy bet (free / $1 / $5 / $10 / $20)
- Optional crypto bet (0.01 / 0.05 / 0.1 / 0.2 ETH via MetaMask)
- Redis queue — players wait until a suitable opponent is found
- Cancel matchmaking at any time

### Battle / Game
- AI-generated problem tailored to average trophy count and chosen language
- Problem difficulty: Easy / Medium / Hard / Very Hard / Expert
- Problem includes: title, description, constraints, hint, test cases
- Time limit set by AI (5 min for Easy up to 90 min for Expert)
- Split-pane layout: problem description (left) + code editor (right)
- Language-aware code editor (textarea with monospace font)
- Submit button runs code against all test cases via Judge0
- Test results shown per test case (pass/fail, runtime, error output)
- Hint button reveals AI-generated hint
- Opponent submission status shown in real-time ("Opponent passed all tests!")
- Timer counts down; color changes yellow at 5 min, red at 1 min
- Fullscreen lock — browser enters fullscreen on match start
- Tab switch detection — first offence = warning, second = disqualification
- Back navigation blocked during active match
- Winner determined by: first to pass all tests → if tie, faster runtime → if tie, earlier submission timestamp
- On timeout: player with passing solution wins; if neither, most test cases passed wins
- Trophy update (+100 win / -100 loss) applied immediately after match
- Badges awarded automatically after win conditions are met

### Crypto Betting (Ethereum)
- MetaMask auto-connect prompt when crypto betting is selected
- Auto-switches MetaMask to Sepolia testnet (adds network if not present)
- Player calls `placeBet(matchId)` on smart contract with ETH value
- Both players must place matching bets before match starts
- On game end, backend calls `settleBet(matchId, winner, loser)` via commission wallet
- Winner receives 98% of total pot; 2% goes to platform commission wallet
- Refund function available if match is cancelled
- Bet status visible in BettingPanel during match
- All transactions confirmed on-chain before state updates

### Leaderboard
- Global leaderboard sorted by trophies (descending)
- Shows: rank, username, trophies, win rate, arena, badge count
- Paginated
- Visible on the game lobby page sidebar

### Arena System
Five arenas based on trophy count:
| Arena | Trophies Required |
|---|---|
| Bronze | 0+ |
| Silver | 1,000+ |
| Gold | 2,000+ |
| Platinum | 3,000+ |
| Diamond | 4,000+ |

### Badge / Achievement System
Badges awarded automatically based on milestones:
- **First Blood** — win your first match
- **Speedster** — solve a problem under 60 seconds
- **Flawless** — win without any failed submissions
- **Win Streak** — win 3 matches in a row
- **Arena Champion** — reach a new arena
- **Bug Hunter** — pass all test cases on first try
- **Comeback Kid** — win after opponent passed tests first
- **Bet Master** — win 5 crypto bets

### Real-time Flow
```
Player A joins queue  →  Redis queue check  →  Match found
       ↓
Match created in MongoDB (problem embedded)
       ↓
Both players receive `match_found` via Socket.io
       ↓
3s countdown  →  `game_start` emitted
       ↓
Players submit code  →  Judge0 executes against test cases
       ↓
`submission_result` sent to submitter
`opponent_submitted` sent to other player
       ↓
All tests pass  →  `game_end` emitted to match room
       ↓
Trophy update  →  Badge check  →  `trophies:updated` + `badges:awarded`
```

### Crypto Betting Flow
```
Player selects crypto bet amount
       ↓
MetaMask connects + switches to Sepolia
       ↓
Player calls placeBet(matchId) on smart contract (sends ETH)
       ↓
Both players' bets confirmed on-chain
       ↓
Game plays out normally
       ↓
Backend calls settleBet(matchId, winner, loser) via commission wallet
       ↓
Smart contract sends 98% to winner, 2% to commission wallet
       ↓
BetSettled event emitted on-chain
```

- Node.js 18+
- MongoDB instance
- Redis instance (RedisLabs or local)
- Judge0 RapidAPI key
- Lyzr AI agent set up with the problem-generator prompt

## Notes for Frontend Redesign

When redesigning the frontend, keep these UX constraints in mind:

1. **Game page is fullscreen during battle** — no scrolling, fixed layout, navbar stays visible at top
2. **Split-pane is mandatory** — problem on left, code editor on right, both equal width
3. **Timer is always visible** — prominently in the header, color-coded
4. **Tab switching is monitored** — warn users visually before they switch
5. **Matchmaking is a blocking state** — user cannot navigate away without cancelling
6. **Crypto betting is optional** — the betting panel only appears if the user opted in before matchmaking
7. **Arena progression is central to the UX** — trophies and arena should be visible on dashboard and lobby
8. **Real-time feedback is critical** — opponent actions (submitted, passed tests) must be shown immediately
